ass drop -- archive session clone(s), or copy generic work upstream

Three modes (disambiguated by argument shape and pwd):

1) Collapse to one session clone (canonical pwd, no args)

Run from the canonical local repo. Archives and removes every session clone
except #1 (newest commit on main). Dirty work from dropped clones rolls into #1.
Prompts for confirmation when more than one clone would be removed.

Usage:
  ass drop

Refuses clones with commits not yet handed off to canonical (unlanded); pushing
canonical to origin is a separate step and does not gate drop.
Refuses clones with an active grok/claude session (quit the session first).

2) Archive and remove session clone #n (canonical pwd)

Index matches ass list / ass status (# column, newest clone = 1). Archives to a
verified .tar.gz first (HARD RULE), then removes the clone directory. Dirty work
rolls into another session clone when one exists.

Usage:
  ass drop <n>
  ass drop <n> --force   # also drop the primary (#1) or your pwd clone

Same refusal rules as mode 1, plus two safety guards:
  - Refuses to remove the primary clone (#1, the 1* active rollover target) or
    the clone at your current pwd unless you pass --force (-f). The active agent
    session is almost always one of these.
  - Dropping the LAST agent session for the repo always prompts and proceeds
    only on an explicit y (even with --force) -- no clones would remain.

3) Copy generic work into agentstartstack (consumer session clone pwd)

Run from a CONSUMER session clone (has .agentstartstack submodule). Copies a file
or directory that belongs upstream in agentstartstack into that repo's newest
session clone -- do not fork it in the consumer.

Usage:
  ass drop <src> [<dest>]

Arguments:
  <src>               file/dir in this clone that belongs in agentstartstack
  <dest>              path relative to the agentstartstack clone root
                      (default: same relative path as <src> here)

Stamps Dropit-Id on single-file drops (when missing) and appends to
.agentstartstack-dropits in the consumer. Then review + commit in the
agentstartstack clone and hand off with ass sync. If <src> was a fork created
here, delete it from this consumer afterward.

Help:
  ass drop help
  ass help drop

See also: docs/ass.md, workflow.md (Dropit + GUID)
