ass new -- create and align a new session clone (from canonical pwd)

Usage:
  ass new
  ass new --grok
  ass new --claude
  ass new [PATH]
  ass new --title "TITLE"

Options:
  --grok              Force Grok / Cursor session clone
  --claude            Force Claude Code session clone
  --title TITLE       Human-set session title shown in 'ass list' (wins over
                      the agent-derived title)

Arguments:
  PATH                Canonical local repo to clone (default: pwd). Accepts '.'
                      or any path; ass new resolves its git toplevel, so it can
                      run from outside the canonical checkout.

Inference order (which agent): grok-only on PATH -> grok; claude-only -> claude;
both installed -> claude. With neither on PATH, ass new exits with an error.

Both agents run in the terminal: start the session by running 'grok' or
'claude' in the session clone that ass new prints. You (human) stay in
canonical and run 'ass' there to land the agent's work.

Run from the canonical local repo (host project or agentstartstack template).
Always creates ~/.ass/worktrees/<project>/<unix-timestamp>/, writes
.agentstartstack.env, aligns clone to canonical. Grok and Claude sessions
share this path (not ~/.grok/worktrees or ~/.claude/worktrees).

Help:
  ass new help
  ass help new
