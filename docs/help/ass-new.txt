ass new -- create and align a new session clone (from canonical pwd)

Usage:
  ass new
  ass new --grok
  ass new --claude
  ass new [PATH]
  ass new --title "TITLE"

Options:
  --grok              Force Grok / Cursor session clone
  --claude            Force Claude Code session clone
  --title TITLE       Human-set session title shown in 'ass list' (wins over
                      the agent-derived title)

Arguments:
  PATH                Canonical local repo to clone (default: pwd). Accepts '.'
                      or any path; ass new resolves its git toplevel, so it can
                      run from outside the canonical checkout.

Inference order (which agent): Codium integrated terminal -> claude; else
grok-only on PATH -> grok; claude-only -> claude; both installed -> claude.
With neither on PATH (and not in Codium), ass new exits with an error.

Codium window: for a claude session, ass new opens a new Codium window on the
clone with the Claude Code extension whenever the codium CLI is on PATH --
regardless of where you launched ass new (you run it from canonical). Opt out
with ASS_NEW_OPEN_CODIUM=0. The window goes to the left monitor, maximized
(needs wmctrl; override origin with ASS_CODIUM_WINDOW_X / ASS_CODIUM_WINDOW_Y).
You (human) stay in canonical and run 'ass' there to land the agent's work.

Run from the canonical local repo (host project or agentstartstack template).
Always creates ~/.ass/worktrees/<project>/<unix-timestamp>/, writes
.agentstartstack.env, aligns clone to canonical. Grok and Claude sessions
share this path (not ~/.grok/worktrees or ~/.claude/worktrees).

Help:
  ass new help
  ass help new
