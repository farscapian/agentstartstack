ass status -- agent session clones vs origin/main and canonical/main

Pwd-oriented: cd to the canonical repo or a session clone, then run ass status.
Read-only: fetches origin/main quietly and counts -- it never local-syncs. Run
'ass sync' yourself to land agent work to canonical. Lists agent session clones
only (not canonical). Reference SHAs appear in the table group-title row.

The canonical 'ahead' column is the count of commits on a clone not yet in
canonical. When any clone has unsynced work (commits ahead of canonical, or
uncommitted/dirty), ass status prints a warning after the table naming each such
clone -- it still does not sync; run 'ass sync' yourself.

Columns (each clone row):
  #       1* = primary (newest) clone (rollover target); ^ = rolls into #1 on
          trim/drop (ass list shows 1,2,3,... for ass drop / ass info index)
  agent   grok / claude
  wip     uncommitted work not in canonical (- or dirty)
  -->     session #1 local-syncs to canonical (ass sync); blank on other rows
  ahead   commits on this clone not on canonical main (group: canonical SHA)
  behind  commits on canonical main not on this clone
  ahead   commits on this clone not on origin/main (group: origin/main SHA)
  behind  commits on origin/main not on this clone
  path    session clone directory (~ shortens $HOME)

Usage:
  ass status             # read-only; use 'ass sync' to land agent work

Help:
  ass status help
  ass help status
