ass -- AgentStartStack handoff CLI

Pwd-oriented: cd to the canonical repo or a session clone, then run a subcommand.

Commands:

  sync                local-sync handoff (session worktree -> canonical)
  adopt               make an agent-created worktree ass-aware (write env + align)
  discover            list agent worktrees for this repo + adopt status
  list                session worktrees for pwd (by origin URL)
  status              ahead/behind for canonical + session clones
  info <n>            plain-language summary for session #n (from ass status)
  drop                archive session clone(s) or copy work upstream (consumer)
  up                  ass sync, then git push origin main
  publish             ass up agentstartstack, then bump .agentstartstack in consumers

Use 'ass <command> help' for detailed help on any command.
Use 'ass help <command>' as an equivalent form (nested topics use extra words,
e.g. ass help sync all, ass help up trim).

Global Options (valid anywhere on the command line):
  -v, --verbose       Show diagnostics (--verbose is a full alias for -v)
  -q, --quiet         Suppress status messages (errors still shown)
  --timestamp         Prefix each line with ISO datetime
  --log-id=ID         Mirror output to ~/.agentstartstack/logs/ass-<ID>.log
                      (implies -v and --timestamp; incompatible with -q)

Repo roots:   $AGENTSTARTSTACK_PROJECT_ROOTS
Session:      ass new -> ~/.ass/worktrees/<repo>/<timestamp> (legacy: ~/.grok, ~/.claude)

See also: docs/cli-help.md (help file layout), docs/ass.md (workflow)
